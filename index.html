This specification defines the 5th major revision of the core language of the World Wide Web: the Hypertext Markup Language (HTML).

    In this version:
  • new features are introduced to help Web application authors.
  • new elements are introduced based on research into prevailing authoring practices.

So, Why should we use

HTML5?

Here's the reasons:

1.HTML5 makes creating accessible sites easier.

2.HTML5 makes your videos and audio truly accessible, you can do many things without third party plugins

3.No more cutting and pasting some long unreadable line of code and no more dirty head tags filled with doctype attributes.

4.HTML5 allows you to write clear and descriptive code, semantic code that allows you to easily separate meaning from style and content.

5.HTML5 allows for storage across multiple windows, it has good security and performance and data will persist even after the browser is closed.

6.HTML5 also comes with a slew of great APIs that allow you to build a better user experience and a beefier, more dynamic web application

7.HTML5 provides a great, mobile friendly way to develop fun, interactive games.

8.HTML5 is being built to make things easier and more cross browser friendly

9.Mobile browsers have fully adopted HTML5 so creating mobile ready projects is as easy as designing and constructing for their smaller touch screen displays — hence the popularity of Responsive Design.

10.The number one reason why you should start using HTML5 today is this: it’s the future, start using it now so you don’t get left behind. HTML5 is not going anywhere and as more and more elements get adopted more and more companies will start to develop in HTML5.
